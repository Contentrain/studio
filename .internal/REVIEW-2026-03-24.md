## Active Findings

### P1 — Security / Data Integrity

#### P1.1 — OAuth state binding incomplete → Phase 4
- Files: `server/api/auth/login.post.ts`, `server/providers/supabase-auth.ts`, `app/pages/auth/callback.vue`, `server/api/auth/verify.post.ts`
- Risk: State token generated but not bound into provider redirect. Callback forwards raw tokens. Login CSRF / session injection possible.

#### P1.2 — GitHub App installation callback → Phase 4
- Files: `server/api/github/setup.get.ts`
- Risk: Trusts `installation_id` from query string. Resolves primary workspace instead of signed context. Replay possible.

#### P1.3 — Project visibility violates access model → Phase 4
- Files: `supabase/migrations/002_fix_rls_recursion.sql`, `server/api/workspaces/[workspaceId]/projects/index.get.ts`
- Risk: Any workspace member sees all projects. Product contract requires explicit project_members assignment for members.

#### P1.4 — Project-member cross-workspace leak → Phase 4
- Files: `server/api/workspaces/[workspaceId]/projects/[projectId]/members/index.get.ts`
- Risk: Route checks owner/admin for provided workspaceId but doesn't verify project belongs to that workspace.

#### P1.5 — Model-scoped authorization not enforced → Phase 4 (EE)
- Files: `server/utils/agent-permissions.ts`, `content/[modelId].get.ts`, `chat.post.ts`
- Risk: `specificModels` / `allowedModels` computed but not enforced on direct reads or chat tool execution.

#### P1.6 — CDN draft content publishing → Phase 6
- Files: `server/utils/cdn-builder.ts`
- Risk: `shouldIncludeEntry()` filtering implemented but publish workflow absent — all entries start as draft with no way to publish. Deferred until publish workflow exists.

#### P1.8 — Branch safety incomplete in chat/diff → Phase 4
- Files: `chat.post.ts`, `content-engine.ts`, `branches/[branch]/diff.get.ts`
- Risk: HTTP merge/reject restrict to `contentrain/*` but chat tool and diff endpoint accept arbitrary branch names.

### P2 — Code Quality / Hardening

#### P2.1 — RLS migration 006 recursion risk → Phase 4
- Files: `supabase/migrations/006_fix_member_rls.sql`, `002_fix_rls_recursion.sql`
- Risk: 006 doesn't drop all 001 policies by name — stale policies accumulate. Needs consolidated RLS migration.

#### P2.2 — Validation materially incomplete → Phase 5
- Files: `server/utils/agent-tools.ts`, `content-validation.ts`, `content-engine.ts`
- Risk: Validate tool is shallow file-presence check. Relation integrity and schema validation not enforced on write path.

#### P2.3 — Conversation routes not scoped to projectId → Phase 4
- Files: `conversations/index.get.ts`, `[conversationId]/messages.get.ts`, `[conversationId].delete.ts`
- Risk: Conversation ownership checked by conversationId + user_id, not by projectId in route.

#### P2.4 — config/vocab bypass protected-branch fallback → Phase 4
- Files: `config.patch.ts`, `vocabulary.patch.ts`, `content-engine.ts`
- Risk: Direct `git.mergeBranch()` instead of Content Engine fallback path. Protected branches leave orphan branches.

#### P2.6 — Workspace settings stale data on slug change → Phase 4
- Files: `app/pages/w/[slug]/settings.vue`
- Risk: Same `onMounted` pattern as P1.9. Needs `watch(slug)` with state reset.

#### P2.7 — Rate limiting not production-grade → Phase 4
- Files: `server/utils/rate-limit.ts`, auth routes
- Risk: In-memory limits, `x-forwarded-for` keyed. Multi-instance/proxy/spoofed headers bypass. Needs Redis or external store.

#### P2.10 — Mobile shell not production-ready → Phase 4
- Files: `app/layouts/default.vue`, project page
- Risk: Hamburger button has no click handler. Mobile-tabs slot empty. Sidebar unreachable on mobile. Needs 2 new organisms + state.
