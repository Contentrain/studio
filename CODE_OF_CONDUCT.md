# Code of Conduct

Contentrain Studio adopts the [Contributor Covenant Code of Conduct v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

This file is a project-specific summary. If any wording here is unclear, the canonical Contributor Covenant text above is the reference.

## Our Pledge

We want participation in the Contentrain Studio community to be respectful, constructive, and safe for everyone, regardless of age, body size, disability, ethnicity, sex characteristics, gender identity or expression, level of experience, education, socioeconomic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Expected Behavior

Examples of behavior that help create a positive environment include:

- Showing empathy and respect toward other people
- Giving and receiving feedback gracefully
- Assuming good intent while staying precise and technical
- Focusing on what is best for the project and its community
- Being respectful of different viewpoints, backgrounds, and experience levels

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, discrimination, or hateful conduct
- Personal attacks, insults, trolling, or deliberately inflammatory comments
- Publishing private information without permission
- Sexualized language or imagery in community spaces
- Sustained disruption of discussions, reviews, issues, or pull requests
- Any conduct that would be inappropriate in a professional setting

## Scope

This Code of Conduct applies in all project spaces, including:

- GitHub issues, pull requests, reviews, and discussions
- Security reports and maintainer interactions
- Project chat, demos, release notes, and other public or semi-public community spaces

It also applies when an individual is officially representing the project.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, contact:

- [security@contentrain.io](mailto:security@contentrain.io)

Please include as much context as possible, including links, screenshots, timestamps, and usernames if available.

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject comments, commits, code, issues, and other contributions that violate these standards.

Possible responses include:

- Private warning
- Temporary moderation or loss of interaction privileges
- Rejection of contributions
- Temporary or permanent ban from project spaces
